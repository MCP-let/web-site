MCPlet よくある質問

MCPletとは何ですか？

MCPletは、Model Context ProtocolとMCP Appsの上に構築されたコードファーストの規約プロファイルです。AI操作に対して明示的な可視性、認証、安全境界を持つビジネスロジックをパッケージ化した、制約付きの単一インテント能力ユニットを定義します。

MCPletはMCPやMCP Appsとどう違いますか？

MCPはツールとリソースのプロトコルを定義します。MCP Appsはホストが UI をレンダリングして調整する方法を定義します。MCPletは両者の上にある規約プロファイルとして、インテントモデリング、ツール分類、可視性ルール、認証要件、ホスト管理の安全境界を追加します。

MCPletの3つのタイプとは何ですか？

MCPletはツールを read、prepare、または action に分類します。read ツールは副作用がなく、自律的な呼び出しに安全です。prepare ツールはコミットメント前に情報を収集または検証します。action ツールは不可逆的な副作用を引き起こすため、より強力な確認と強制が必要です。

MCPletはコードファーストですか、YAMLベースですか？

主要プロファイルはコードファーストです。MCPletのメタデータは、_meta.mcpletType、_meta.visibility、_meta.auth、_meta.ui などのフィールドを使用してツール登録時に宣言されます。

mcplet.yaml は後方互換性のデフォルトとして引き続き使用できますが、コードメタデータが存在する場合は唯一の真実の情報源ではありません。

MCPletはパスキー認証をどのように使用しますか？

保護されたMCPletは _meta.auth で認証を宣言します。モデルに見えるアクションツールの場合、ホストは呼び出しをインターセプトし、ブラウザコンテキストを通じてパスキーアサーションを取得し、ビジネス引数の外に検証マテリアルを注入します。MCPletバックエンドはアクションを実行する前にアサーションを検証します。

MCPletはどのホストプロファイルをサポートしますか？

MCPletは現在2つのホスト実装プロファイルを定義しています。WebUI Profile はMCP Apps レンダリングを持つMCPクライアントまたはエージェントシェルを対象とします。Agent Profile は専門エージェントと外部設定LLMで構成されるオーケストレーションシステムを対象とし、汎用UI層は不要です。

MCPletを始めるにはどうすればよいですか？

ホストプロファイルを選択し、各ツールを read、prepare、または action として分類し、_meta にコードファーストのメタデータを登録し、結果スキーマを公開し、保護されたモデルに見えるアクションにパスキー強制を追加することから始めてください。

最短経路を望むなら、まず はじめにガイド を使用してから、完全な仕様に対して詳細を検証してください。
