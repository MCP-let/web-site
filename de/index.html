Kernkonzepte

MCPlet verstehen

MCPlet ist die kleinste bereitstellbare, prüfbare, KI-bedienbare Einheit und repräsentiert genau eine Geschäftsabsicht.

◈

Einzelne Absicht

Jedes MCPlet repräsentiert genau eine Geschäftsabsicht. Keine Mehrdeutigkeit, keine Komplexität, sondern fokussierte, zweckgerichtete Fähigkeit.

⚙

MCP-Tool-Integration

Jedes MCPlet kapselt einen MCP-Tool-Aufruf und bietet eine standardisierte Schnittstelle für KI-Systeme.

📱

Optionale UI

Bietet reichhaltige interaktive Erlebnisse über MCP Apps, mit automatischem Fallback auf textbasierte Antworten.

🔒

Sicherheit zuerst

Explizite Lebenszyklus- und Sicherheitsbeschränkungen stellen sicher, dass KI-Systeme innerhalb definierter Grenzen arbeiten.

📊

KI-Metadaten

Reichhaltige Ausgabemetadaten ermöglichen KI-Überlegungen zu Operationen, Vorschlägen und nächsten Schritten.

🚀

Stateless-Design

Reines Funktionsmodell ohne versteckten Zustand. Alle Daten fließen über den Host für Transparenz.

🔒

Passkey-Authentifizierung

Integrierte FIDO2-Passkey-Unterstützung für sichere, passwortlose Authentifizierung mit direkter AI-Chat-Integration für schnelles und einfaches Human-in-the-loop.

Klassifizierung

MCPlet-Typen

Jedes MCPlet deklariert genau einen Typ und definiert Sicherheitsprofil und angemessene Nutzung.

read

Abfrage und Suche

Sichere, idempotente Operationen zur Datenabfrage ohne Nebenwirkungen.

  • Keine Nebenwirkungen
  • Idempotent durch Design
  • Sicher für autonome KI
  • Gibt angereicherte Daten mit Metadaten zurück
prepare

Validierung und Vorbereitung

Informationen sammeln oder validieren, bevor irreversible Aktionen ausgeführt werden.

  • Kann Bestätigung erfordern
  • Keine irreversiblen Nebenwirkungen
  • Gibt strukturierte Vorbereitungsdaten zurück
  • Ermöglicht schrittweise Workflows
action

Ausführen und Bestätigen

Operationen mit irreversiblen Nebenwirkungen und verpflichtender menschlicher Aufsicht.

  • Explizite menschliche Bestätigung
  • Klare Absicht in der UI
  • Gibt Ergebnisse mit Anleitung zurück
  • Vollständige Auditspur wird geführt
Sicherheit

Passkey-Authentifizierung

Integrierte FIDO2-Passkey-Unterstützung für sichere, passwortlose Authentifizierung mit direkter AI-Chat-Integration für schnelles und einfaches Human-in-the-loop.

_meta contract
{
  "_meta": {
    "mcpletType": "action",
    "visibility": ["model", "app"],
    "mcpletToolResultSchemaUri": "mcplet://tool-result-schema/approve_order",
    "ui": {
      "resourceUri": "ui://orders/approve.html",
      "displayMode": "inline"
    },
    "auth": {
      "required": "passkey",
      "enforcement": "strict",
      "promptMessage": Bitte authentifizieren Sie sich mit Passkey, um den Vorgang zu bestätigen
    }
  }
}
  • 🔒

    FIDO2-Standard

    Verwendet WebAuthn/FIDO2-Standards für maximale Sicherheit und Browser-Kompatibilität.

  • 📱

    Keine Passwörter

    Passwortlose Authentifizierung eliminiert Credential-Stuffing- und Phishing-Angriffe.

  • ⚙

    Flexible Konfiguration

    Einfache YAML-Konfiguration mit anpassbarer Server-URL und Authentifizierungsaufforderungen.

  • 🔒

    Browser-Nativ

    Direkte Integration mit nativen Passkey-APIs des Browsers für nahtloses Benutzererlebnis.

Designprinzipien

MCPlet-Philosophie

Auf fünf Kernprinzipien aufgebaut, die KI-Interaktion verlässlich, vorhersehbar und prüfbar machen.

01

Einzelne Absicht

Jedes MCPlet erledigt genau eine Sache. Kein Schweizer Taschenmesser, sondern fokussierte, zweckgerichtete Fähigkeit.

02

Standardmäßig stateless

Kein versteckter Zustand; alle Daten fließen über den Host. Transparenz ist Designanforderung.

03

KI-First-Design

Die Ausgabe enthält speziell für KI-Denken und Entscheidungsfindung entwickelte reichhaltige Metadaten.

04

Fortschrittliche Verbesserung

Funktioniert im Textmodus vollständig und wird bei MCP-Apps-Unterstützung durch eine rich UI erweitert.

05

Sicherheitsbeschränkungen

Explizite Nebenwirkungen, verpflichtende Prüfung für Aktionen. Sicherheit ist Designprinzip.
